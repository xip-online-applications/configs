# Configs

[![Publish](https://github.com/xip-online-applications/configs/actions/workflows/publish.yml/badge.svg)](https://github.com/xip-online-applications/configs/actions/workflows/publish.yml)

This repository houses XIP's config files, all publicly available as [npm packages](https://www.npmjs.com/org/xip-online-applications).

## How to release

These configs are released using [Lerna](https://lerna.js.org/) for versioning (changelog entry & package.json) and publishing. Published as npm packages using GitHub Actions trusted publishing.

Before the workflow can publish, each public package on npm must have a trusted publisher configured for:

- GitHub organization or user: `xip-online-applications`
- Repository: `configs`
- Workflow filename: `publish.yml`
- Environment name: leave empty unless the workflow uses a protected GitHub environment

The "publish" workflow uses GitHub OIDC and no longer requires an `NPM_PUBLISH_TOKEN` secret.

### Conventional commits

You should be using [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/) for your commit messages. With these conventions, Lerna can generate proper changelog entries and update the version accordingly.
